# AI 工具越用越多，Skills 乱套了怎么办？

> **目标读者**：有 AI 工具使用经验，Skills 数量 10+
> **核心钩子**：62 个技能、3 个工具、N 个设备，我是怎么理清楚的
> **字数目标**：1200-1500 字

---

## 一、问题：工具越多，越乱

你有没有遇到过这些问题：

- **在公司电脑改了一个 Skill，回家打开个人 Mac，发现还是旧版本？**
- **同一个功能，你在 Cursor、Claude Code、Kiro 里建了三个 Skills，每次改都要改三遍？**
- **同事问你借个 Skill，你找不到最新的版本在哪？**
- **想回滚到上周的版本，发现只有 Git 记录，文件本身已经改乱了？**

这几个破事让我全赶上了。

**Skills 越来越多，越来越乱。**

简单说，Skills 就是给 AI 工具的「专属指令包」。你告诉 AI：遇到这类任务，按这个流程来。比如我有一个 `indicator-search`，专门用来在公司指标平台搜索业务指标；有一个 `start-my-day`，每天早上帮我规划当天的工作。

这些 Skills 把 AI 从通用助手变成了懂我业务的专属工具。

---

## 二、诊断：三个核心矛盾

先把问题想清楚。

**矛盾1：数量 vs 管理**

Skills 从 10 个涨到 62 个，管理方式还是「手动复制文件」。用 Excel 管理 100 人的项目——不是不能用，但迟早出问题。

**矛盾2：专用 vs 通用**

我有 36 个工作专用 Skills（公司内部系统的），14 个通用开发 Skills，12 个知识管理 Skills。混在一起，找起来很费劲。

**矛盾3：多工具共用**

Kiro、Claude Code、Cursor 格式基本兼容，但各自存在不同目录。同一个 `brainstorming` Skill，我有三个版本，内容还不一样。

最烦的是：**每次换设备，都要重新手动拷贝一遍。**

---

## 三、解法：统一技能库 + 软链接分发

解法其实不复杂：

**建一个 Git 仓库作为「单一真相来源」，用软链接把技能分发到各工作区。**

```
~/skills-library/          ← 唯一的真相来源（Git 管理）
├── 01-连接/               ← 认证、API 通道
├── 02-感知/               ← 搜索、探查、理解数据
├── 03-分析/               ← 诊断、审计、验证
├── 04-创造/               ← 生成文档、写入系统
├── 05-思维/               ← 规划、研究、日常流程
├── 06-工程/               ← 开发质量、调试、Git
├── 07-知识/               ← 格式规范、知识载体
└── 08-元/                 ← 技能管理自身
```

分类逻辑是**按能力层级**，从底层基础设施到上层创造输出：

> 连接 → 感知 → 分析 → 创造 → 思维 → 工程 → 知识 → 元

然后让 AI，把技能库里的每个 Skill 软链接到对应的工作区目录。
