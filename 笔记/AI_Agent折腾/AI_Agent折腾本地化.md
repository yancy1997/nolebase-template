# AI Agent 折腾本地化

## 背景

用 AI Agent 把"折腾"变成对话驱动型，建立选题库，想整哪个整哪个，无顺序无压力。折腾的本质是解决实际问题，AI Agent 极大提升了折腾效率。

## 核心洞察

1. **全流程对话驱动**：Agent 替代搜索→复制→粘贴→调试的全过程
2. **触发器复杂度是真正痛点**：写脚本不难，难的是怎么优雅地触发它（HA、Siri、定时任务各有门槛）
3. **本地自整方案足够好**：不追求通用解，只解决自己环境的问题
4. **Agent 降低了定制化成本**：以前定制化需要写代码，现在对话即可

## 可折腾的设备/服务

| 设备/服务 | 状态 | 说明 |
|-----------|------|------|
| 飞牛 NAS（fnOS） | 已验证 | 开关机命令 WOL + SSH |
| OpenWrt 路由器 | 待规划 | 管理设置、开关服务 |
| HomeAssistant | 待规划 | 智能家居控制 |
| Docker 服务 | 待规划 | 部署管理 |
| Nginx 反向代理 | 待规划 | 域名访问本地服务 |

| OECT 小主机 | 待规划 | 更多玩法探索 |

## 技术方案

### 飞牛 NAS 控制

**唤醒（WOL）：**
```bash
wakeonlan 24:4B:FE:98:D7:BE
```

**关机（SSH）：**
```bash
sshpass -p '密码' ssh -i ~/.ssh/id_ed25519_feiniu -o StrictHostKeyChecking=no yancy@192.168.100.148 "/sbin/poweroff"
```

### 触发词

启动飞牛 / 关闭飞牛 / 启动 NAS / 关闭 NAS / 启动 bas / 关闭 bas

## 价值总结

- **传统方案**：搜索帖子 → 复制命令 → 调试脚本 → 配置触发器（HA/Siri/定时）→ 适配不同环境
- **AI Agent 方案**：对话即执行，本地自整，不需要通用解

## 延伸方向

- 扩展到状态查询、备份触发、服务重启
- 与 cc-connect 联动实现更自然的语音触发
- 定时任务触发
